GalleryApp
Якщо перейти по силці galleryapp/, то нам відкриється головна сторінка, на якій ми побачимо додані відео або фото, які ми додали по силці 
galleryapp/add/. На цій сторінці ми бачимо, що можна додати назву, опис, вибрати тип фото чи відео, а нижче ми можемо це фото або відео 
завантажити. У редакторі коду, відкривши папку galleryapp і відкривши файл models.py, ми побачимо модель GalleryItem. У ній будуть такі 
поля, як: title (Назва матеріалу), description (Опис матеріалу), media_type (Тип медіа: photo або video), file (Завантажений файл: 
зображення або відео), uploaded_by (Користувач, який завантажив файл), is_approved (Чи затверджено модератором/адміністратором) і 
uploaded_at (Дата завантаження). Далі, якщо подивитися в папку templates/galleryapp, ми побачимо наші шаблони, такі як gallery_list.html 
(відображення всієї галереї — сітка карток) і gallery_add.html (форма для додавання нових матеріалів). У файлі views.py ми можемо побачити 
такі в'ю: gallery_list (це в'ю відображає всі фото і відео у галереї), gallery_add (дає нам можливість додавати нові фото або відео до галереї), 
gallery_moderate (доступна лише адміністраторам для перевірки нових матеріалів).
MaterialsApp
Коли ми переходимо по силці materialsapp/, ми бачимо головну сторінку. На ній ми бачимо список матеріалів. Ми можемо видалити створені 
матеріали або редагувати їх. Також ми можемо додати новий матеріал. Натиснувши на кнопку "Додати новий матеріал", ми бачимо, що у нас є поле 
для назви і опису матеріалу. Також ми маємо обрати тип матеріалу, тобто файл, зображення, YouTube посилання або зовнішнє посилання. Нижче ми 
можемо завантажити файл із зображенням, і є поля для посилання на YouTube і просто посилання. І нижче є кнопка "Назад до списку", яка поверне нас 
на головну сторінку з іншими доданими матеріалами.
Відкривши папку materialsapp і файл models.py, ми побачимо нашу модель Material. У ній є такі поля: title (назва матеріалу), description (опис 
матеріалу), material_type (тип матеріалу), file (файл), image (зображення), youtube_link (YouTube посилання), external_link (зовнішнє посилання), 
created_by (автор), created_at (дата створення), updated_at (дата оновлення).
У папці templates/materialsapp ми бачимо такі шаблони: materials_list.html (сторінка зі списком усіх матеріалів), material_add.html (форма 
додавання нового матеріалу), material_edit.html (форма редагування існуючого матеріалу), material_confirm_delete.html (підтвердження видалення 
матеріалу).
Якщо ми перейдемо у файл views.py, то побачимо такі в'ю: material_list (список матеріалів), material_create (створення матеріалу), material_edit 
(редагування матеріалу), material_delete (видалення матеріалу).
